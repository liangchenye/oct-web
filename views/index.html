The open container testing framework.
Aims to promote open container initiative

Scopes

The open container testing framework covers 4 parts:

Verify if a image or a runtime was complicant with the OCI standard.

Test the io/network and other benchmark of running a container.

Make sure if a container runtime was fragile to the security attack.

Better understand the function of each container.

Architecture

The whole OCT including three main components.

Tools: open source tools used to test the most popular container products.

Case: Well defined specification with json format. Any case developers could write his/her own test case, either by his/her own tool or by the 'OCT tools'.

Engine: micro service, receive testcase, deploy to the according host operation system and container and run it. Collect and generate a readable test report.

Tools

Bundle Validator: verify if a image/bundle was a standard container.

Runtime Validator: verify if a runtime was compliant with OCI standard.

Cadvisor/iperf: performance tools from community.

Cases

TheCase is easy to develop and maintain.

Using the 'case generator' tool and 'case validator' tool to make life eaiser.

Using the official casedb or maintain own db. are both OK.
